## 4. Example User Journey: "The Off-By-One"
- **User Action:** Submits a Binary Search algorithm that loops infinitely if the target isn't found.
- **Solver Output:** Fails to visually identify the index boundary flaw.
- **Reflection Output:** Catches that `left <= right` boundary limits are mismatched with `right = mid`. Flags the code for rewrite.
- **Verification Output:** Simulates an array bounds call `search([1,2,3], 4)`, confirming the infinite loop. Rewrites boundary logic.
- **QA Polished Result:** User is presented the fixed code with an extremely supportive message explaining *why* binary search boundaries crash.

## 5. Documented Failure Cases & Trust Affirmation
We do not hide our limits. CodeMentor AI **fails** under these parameters:
1. **Extensive Graph Theory (OOM errors):** Because the Verification Proxy is a "mental dry-run" simulating execution without standard Python `stdout`/`stderr` stacks, it cannot reliably calculate depth limits on complex trees, sometimes allowing stack overflows.
2. **Obscure Math Papers:** If asked to solve un-documented discrete math queries (e.g. obscure number theory beyond LeetCode scopes), the Reflection loop will oscillate entirely.

**Current Mitigation:** The QA agent handles oscillating outputs gracefully, stating: *"This algorithm contains edge cases that exceed the simulated verification bounds. Manual runtime review is advised."*
